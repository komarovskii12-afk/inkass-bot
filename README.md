# Бот учёта инкассации изношенной валюты

<!-- autodeploy test -->


Телеграм-бот для контроля приёма изношенной («неликвидной») валюты с обменных пунктов.
Работник фиксирует, сколько купюр принято с каждого пункта по валюте и номиналу,
сколько из них оказалось в нормальном состоянии и сколько взято в работу.
Владелец смотрит сводку за любой день кнопкой в самом боте.

## Что умеет

- **📥 Принять инкассацию** — выбор даты → пункта → валюты/номинала → ввод чисел
  (принято / годных / в работу). Можно добавить несколько номиналов в одну приёмку.
- **📊 Отчёт** — выбираешь день, бот присылает сводку по пунктам с итогами (штуки и суммы в $).
- **🏢 Пункты** (только владелец) — добавить пункт, включить/выключить (скрыть) пункт.
- Доступ по белым спискам ID: работники вводят данные, владельцы видят всё.

Валюты и номиналы сейчас: **USD 100 и 50**. Добавляются в `config.py` (`CURRENCIES`)
без изменения остального кода.

## Шаг 1. Создать бота в Telegram

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → задай имя.
2. Скопируй **токен** (`BOT_TOKEN`).
3. Узнай свой числовой ID: напиши [@userinfobot](https://t.me/userinfobot) — это твой `ADMIN_IDS`.
   Пусть каждый работник тоже напишет ему и пришлёт тебе свой ID — это `WORKER_IDS`.

## Шаг 2. Локальный запуск (по желанию, для проверки)

```bash
pip install -r requirements.txt

# зададим переменные (Linux/Mac):
export BOT_TOKEN=токен
export ADMIN_IDS=твой_id
export WORKER_IDS=id1,id2
export DEFAULT_POINTS="Центр,Вокзал"

python bot.py
```

Без `BASE_WEBHOOK_URL` бот работает в режиме polling и хранит данные в локальном
файле `bot.db` (SQLite). Для боевого режима — см. ниже.

## Шаг 3. Деплой на Render (бесплатно)

Вариант проще — через Blueprint (`render.yaml` уже в проекте):

1. Залей эти файлы в репозиторий GitHub.
2. На [render.com](https://render.com) → **New → Blueprint** → выбери репозиторий.
   Render прочитает `render.yaml`, создаст веб-сервис **и** базу PostgreSQL.
3. В процессе Render попросит заполнить переменные, у которых `sync: false`:
   - `BOT_TOKEN` — токен от BotFather
   - `ADMIN_IDS` — твой ID
   - `WORKER_IDS` — ID работников через запятую
   - `DEFAULT_POINTS` — стартовые пункты через запятую (можно оставить пустым)
4. Нажми **Apply**. Остальное (`DATABASE_URL`, `BASE_WEBHOOK_URL`, `WEBHOOK_SECRET`)
   подставится автоматически.

После деплоя бот сам установит webhook и заполнит список пунктов. Напиши боту `/start`.

> Ручной вариант (без Blueprint): создай **Web Service** (Build: `pip install -r requirements.txt`,
> Start: `python bot.py`) и отдельно **PostgreSQL**. В переменные веб-сервиса добавь всё из
> `.env.example`, где `BASE_WEBHOOK_URL` = адрес сервиса (`https://имя.onrender.com`),
> а `DATABASE_URL` — Internal Database URL из настроек базы.

## Важно про бесплатный Render

- Бесплатный веб-сервис «засыпает» без трафика. Бот работает через **webhook**, поэтому
  просыпается сам при входящем сообщении (первое сообщение после сна может дойти
  с задержкой в несколько секунд).
- Бесплатная база PostgreSQL на Render имеет ограниченный срок жизни — следи за
  уведомлениями Render и делай бэкап (кнопка экспорта в панели базы), либо перейди
  на платный план, когда бот пойдёт в реальную работу.

## Как пользоваться

1. `/start` — открывается меню.
2. **Принять инкассацию**: дата → пункт → номинал → принято → годных → в работу →
   «Ещё номинал» или «Завершить».
3. **Отчёт**: выбери день → получишь сводку.
4. **Пункты** (владелец): добавляй точки и скрывай неактуальные.
